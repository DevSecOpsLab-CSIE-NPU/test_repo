# ch01 Git 特別檔案

## gitignore

### 功能解說

.gitignore 檔案是 Git 版本控制系統中的一個特殊檔案，用於指定 Git 應該忽略的檔案或目錄，這些被忽略的項目不會被 Git 追蹤。

### 主要用途

- 防止敏感資訊（如密碼、API 金鑰）被意外提交
- 排除編譯產生的檔案或目錄
- 忽略特定於開發環境的檔案
- 減少儲存庫大小，提高效能

### 使用方法

1. 在專案根目錄創建 .gitignore 檔案
2. 在檔案中列出要忽略的檔案、目錄或模式，每行一個
3. 使用 # 符號添加註釋
4. 使用 * 作為萬用字元
5. 使用 ! 來否定一個模式

### 常見模式範例

```nasm
# 忽略所有 .log 檔案
*.log

# 忽略 build 目錄
/build/

# 忽略 .env 檔案，但不忽略 .env.example
.env
!.env.example

# 忽略所有 .txt 檔案，除了 doc 目錄下的
*.txt
!/doc/*.txt
```
